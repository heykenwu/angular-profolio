Steganography application in Python

Steganography is the art and science of the communicating in a way that conceals any existence of the communication of information. The technique involves concealing the actual information within a "carrier", which can be image, video, or audio file. It is implemented in a manner that prevents a third party from detecting the existence of the concealed message or file. In this project, we are going to design a simple LSB stego application written in Python. The application will be command-line , with the appropriate switches to perform the various functions.

Approach

There are two main functions of this application. First one is to encrypt the secret image by password using DES/3DES encryption and hide the bits of cipher data to the least significant bits in every R, G, B component of the cover pixels. Second one is to extracting bits from the cipher image and decrypted them by password using DES/3DES to the original data and recover the data to the original secret image. Application Structure

Library that used in the Application (prerequisites):

  • Python des

  • Python PIL

The application has 3 modules: dcstego.py, dcimage.py and decutils.py. They are responsible for different part of the functions of the program.

  • dcstego.py: the main functions that contains general functionality such as parsing command line arguments, check the options (encryption or decryption), check the data size and format of both cover image and secret image as well as the validation of the key. In this module, we use ‘Argparse’, which is a recommend command line parser for python to get the information about the images from the user.

  • dcimage.py: the module that contains the image processing like changing the image to the binary string or to hex values, and to manipulate the image like changing its LSB.

  • decutils.py: the module will contain four functions. There are for the encryption or decryption of the header or the data of secret image.

Encryption used in this application:

DES/3DES are used in this application depends on the length of your key (8 chars will be DES and 16 bytes will be 3DES). Since the block size of DES is 8 bytes, the encrypted header size will be the multiple of 8. If the length of the data size can not be divisible by 8, padding will be added to the data to make the length be the multiple of 8. For example, if the size of header is 11, the output size will be 16. And the data of the encryption will be added after the encrypted header. The idea is shown below:

figure 8 encrypted data structure

To get the data of the image, you need to know the data size, which in order to get it, you will need to first decrypt the header.
